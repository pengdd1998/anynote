## Phase 80.1: Wiki-Style Note Linking

### Implementation Summary

Implemented bidirectional wiki-style [[note links]] for local note connections. Pure client-side data, no server sync.

### Architecture Decisions

**Fat Client, Thin Server**:
- All note link operations go through local SQLite via NoteLinksDao
- No server API calls for link CRUD
- Sync engine excludes note links from push/pull cycles

### Files Created

**Frontend:**
- `lib/features/notes/presentation/embeds/wiki_link_embed.dart` — Custom Quill embed builder for rendering [[links]] as tappable widgets
- `lib/features/notes/presentation/widgets/wiki_link_picker_sheet.dart` — Bottom sheet for searching and selecting notes when typing [[
- `lib/features/notes/presentation/widgets/backlinks_sheet.dart` — Shows inbound links (notes that link TO this note)
- `lib/features/notes/presentation/widgets/related_notes_sheet.dart` — Shows outbound links (notes this note links TO)
- `lib/core/database/daos/note_links_dao.dart` — Local DAO for CRUD operations

### Files Modified

**Frontend:**
- `lib/core/database/tables.dart` — Added NoteLinks table (id, sourceId, targetId, linkType, createdAt)
- `lib/core/database/app_database.dart` — Schema v9, migration v8→v9 adds NoteLinks table with indexes
- `lib/features/notes/presentation/note_editor_screen.dart` — [[ pattern detection, Related Notes button in app bar
- `lib/features/notes/presentation/rich_note_editor.dart` — WikiLinkEmbedBuilder added to embedBuilders
- `lib/l10n/app_*.arb` — Added localization: wikiLink, linkToNote, relatedNotes, backlinks, etc.

### Database Schema

```sql
CREATE TABLE note_links (
  id TEXT PRIMARY KEY,
  source_id TEXT REFERENCES notes(id),
  target_id TEXT REFERENCES notes(id),
  link_type TEXT DEFAULT 'wiki',
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_note_links_source_id ON note_links (source_id);
CREATE INDEX idx_note_links_target_id ON note_links (target_id);
```

### Key Features

- Typing `[[` in editor triggers searchable note picker
- Clicking [[link]] navigates to target note
- Backlinks panel shows which notes reference current note
- Related notes panel shows outbound links
- Local-only storage (deleted when note is deleted)
- No server sync required

## Phase 81: Graph View for Note Connections

### Implementation Summary

Enhanced the note graph visualization with a force-directed layout algorithm, interactive pan/zoom support, and local data integration using NoteLinksDao from Phase 80.1.

### Key Features

- **Force-directed layout**: Nodes self-organize using repulsion forces between all nodes and spring forces along edges
- **Interactive pan/zoom**: Pinch-to-zoom (0.5x to 3x) and pan support for exploring large graphs
- **Hover effects**: Nodes highlight when hovered, with enlarged size and border
- **Local data**: Uses `localGraphDataProvider` with NoteLinksDao (no API calls)
- **Empty state**: Helpful message when no notes exist
- **Error handling**: Graceful error display with retry capability

### Files Modified

**Frontend:**
- `lib/features/notes/presentation/widgets/note_graph_screen.dart` — Complete rewrite with:
  - `localGraphDataProvider` — Uses NoteLinksDao instead of API
  - `GraphData` dataclass for nodes and edges
  - `_GraphCanvas` — Interactive canvas with force-directed layout
  - `_GraphPainter` — Custom painter with shadows, labels, and hover effects
- `lib/features/notes/presentation/notes_list_screen.dart` — Added graph button (account_tree_outlined icon)

### Force-Directed Layout Algorithm

```dart
// Parameters
_repulsion = 50000        // Node-to-node repulsion
_springLength = 120       // Desired edge length
_springK = 0.05          // Spring constant
_damping = 0.85          // Velocity damping
_maxIterations = 300     // Simulation steps
```

The algorithm:
1. Initializes nodes in a circle
2. Iteratively applies repulsion forces between all node pairs
3. Applies spring forces along connected edges
4. Updates positions with velocity damping
5. Constrains nodes within canvas bounds
